# SQL Performance Tuning

SQL practice files from **Section 8 – Performance Tuning of SQL Queries** of the Udemy Data Engineering course.

## Topics Covered (Section 8)
- Creating and using indexes effectively
- Understanding EXPLAIN and EXPLAIN ANALYZE output
- Query cost analysis and execution plans
- Avoiding full table scans
- Optimising JOIN performance
- Partitioning strategies
- Avoiding N+1 query problems

## Files in This Folder
| File | Description |
|------|-------------|
| `indexes.sql` | Creating, analysing and dropping indexes |
| `execution_plans.sql` | Reading EXPLAIN ANALYZE output |
| `query_optimisation.sql` | Rewriting slow queries |
| `join_optimisation.sql` | Improving JOIN performance |

## Key Takeaways
- Indexes speed up reads but slow down writes — use them strategically
- Always EXPLAIN your slow queries before guessing at fixes
- Avoid SELECT * in production queries
